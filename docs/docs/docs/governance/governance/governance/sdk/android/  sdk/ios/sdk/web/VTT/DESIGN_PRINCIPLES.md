# VTT Design Principles (066–100)

**Version:** 1.0
**Owner:** Jason Wineti (Ngāi Tahu)
**Status:** Public Technical Surface | Safe to Publish

---

These principles are the architectural laws of the Unified Platform. They describe how the system behaves, not how it is implemented. The underlying algorithms remain sealed.

---

## The Principles

066. The Identity Singularity Principle — A platform becomes sovereign when identity, permissions, and context collapse into a single, verifiable layer.
067. The Trust Agent Law — AI must change its behaviour based on who it is speaking to, not what they ask. Mode > Prompt.
068. The Household Gravity Effect — The more daily tasks the platform handles, the stronger the gravitational pull on the household.
069. The Youth Safety Paradox — Kids want freedom. Parents want control. The platform must satisfy both simultaneously.
070. The BRICK Loop — Reward → Action → Progress → Reward. A closed motivational loop that powers the youth economy.
071. The Sovereign Stack Rule — Identity + Governance + Attestation = Sovereign-grade infrastructure.
072. The Context Mesh — Every module shares context with every other module. This creates exponential intelligence.
073. The Parent Override Doctrine — Parents must always have the final say — but the system should rarely force them to use it.
074. The Brand Safety Covenant — Brands can engage youth, but never target them. Engagement must be earned, not extracted.
075. The Creator Horizon — Teens must be able to build inside the platform, not just consume. Creation = retention.
076. The Calendar Spine — Time is the backbone of the Family OS. Everything attaches to the calendar.
077. The Grocery Wedge — The fastest path into the home is through the pantry. Solve groceries → win the household.
078. The Trust Multiplier — Every safe interaction increases trust. Every unsafe interaction destroys it. Trust compounds or collapses.
079. The Mode Integrity Rule — Kid Mode must never leak into Teen Mode. Mode boundaries are sacred.
080. The Zero-Data Engagement Model — The platform must generate value without personal data extraction.
081. The Household AI Trinity — Meals → Money → Time. Solve these three and you own the household OS.
082. The Youth Economy Firewall — Kids can earn, save, and spend — but never borrow, invest, or risk.
083. The Brand Quest Law — A quest must be fun, fair, safe, rewarding, and parent-approved.
084. The Creator-Brand Bridge — Teens should collaborate with brands safely, without personal data exposure.
085. The Household AI Persona Rule — The AI must feel like part of the family, but never replace a parent.
086. The Grocery Intelligence Loop — Shopping → Pantry → Meals → Budget → Shopping. A closed optimisation loop.
087. The Education Safety Line — AI can teach. AI can guide. AI cannot grade, diagnose, or judge.
088. The Wellness Boundary — Wellness AI must never cross into medical or psychological territory.
089. The Sovereign Adoption Curve — Governments adopt identity systems slowly, then all at once.
090. The Household Lock-In Effect — Once a family uses 4+ modules, churn approaches zero.
091. The Teen Autonomy Threshold — Teens need increasing control, but within a governed sandbox.
092. The Brand Flywheel — More families → more brands → more quests → more BRICK flow → more families.
093. The Creator Economy Catalyst — Give teens tools → they create content → brands sponsor → platform grows.
094. The Trust OS Halo — Trust OS elevates every module. Every module reinforces Trust OS.
095. The Household AI Convergence — All household tasks converge into time, money, energy, and attention. The platform optimises all four.
096. The Parent Confidence Index — Parents stay when the AI is predictable, rules are clear, controls are simple, and outcomes are positive.
097. The Youth Engagement Law — Kids engage for fun. Teens engage for autonomy. Parents engage for utility.
098. The Sovereign-Ready Principle — A platform is sovereign-ready when it can operate without ads, data extraction, or external dependencies, with verifiable governance.
099. The Unified Platform Effect — When V1 and V2 merge, the platform becomes more valuable than the sum of its parts.
100. The Immutable Truth — Trust is the product. Everything else is the ecosystem.
